# 🏥 Patient-information (患者説明資料リポジトリ)

本リポジトリは、耳鼻咽喉科・頭頸部外科をはじめとする診療現場でのインフォームド・コンセント（IC）を支援するため、分かりやすい患者説明用の医療文書を構造化・管理する場所です。

## 📋 概要
- **目的**: 診療現場での疾患説明や手術術式説明の品質平準化、および平易で的確な説明情報の提供。
- **管理言語**: 主に日本語（多言語対応用のソースも含む）。

## 📂 ディレクトリ構成
- **`src/`**: 患者説明資料のMarkdown原本ファイル（50件以上を蓄積）。
- **`HTML/`**: 自動生成されたHTMLプレビュー・印刷用ファイル（※Git管理外）。
- **`generate_html.py`**: Markdownから印刷およびスライド表示対応のHTMLへ自動変換するビルドスクリプト。
- **`.gitignore`**: ビルド成果物である `HTML/` ディレクトリやIDE設定等を除外。

## ⚙️ 運用方針
1. **SSOT（信頼できる唯一の情報源）の原則**:
   - `src/*.md` に格納されている Markdown ファイルを正（原本）として管理します。
2. **生成物の同期管理**:
   - ガイドラインの当面の方針に従い、原本と生成されたHTML/PDFのバージョン（世代）を完全に一致させるため、ビルドされたHTMLファイル群も含めてすべて同一リポジトリでGit管理します。
3. **個人健康情報（PHI）の防護**:
   - `ObsidianVault/rules.md` に従い、特定の患者データやプライバシーに関わる情報をリポジトリ内に一切含めないように徹底します。


## 🚀 ビルド手順
ローカルでHTMLファイルを生成してブラウザプレビュー・印刷を行いたい場合は、以下のコマンドを実行します。

```bash
python generate_html.py
```

実行後、`HTML/` ディレクトリ内に各Markdownに対応する `.html` ファイルが生成されます。
生成されたHTMLは「通常画面表示」「高齢者向け印刷」「紙節約印刷」「スライド表示」の切り替えが可能です。
